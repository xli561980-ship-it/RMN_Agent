# Demo Script

这份文档用于面试、作品集展示或售前风格演示。实际讲解时，可以根据你放入 `data/papers/` 和 `data/manuals/` 的本地非敏感文档调整示例问题。

## 30 秒项目介绍

RMN Agent 是一个面向实验室知识工作的 AI Agent Demo。它可以让用户围绕科研论文和 SOP / 手册文档提问，同时在系统设计上区分“研究证据”和“操作规范”两类知识来源。这个项目的重点不是做一个普通聊天机器人，而是展示一个可运行的 AI 解决方案：包含文档入库、意图路由、证据检索、带引用回答生成，以及一个用于演示的 Streamlit 界面。

从售前视角看，这个项目展示了我如何把业务痛点转化为可演示方案：用户需要快速从复杂技术文档中获得答案，但涉及安全、实验步骤或操作建议时，回答必须能追溯到手册或 SOP。

## 2 分钟 Demo 讲解

1. **介绍场景**

   “假设一个科研或 R&D 团队有很多论文、补充材料、设备手册和内部 SOP。用户经常会问：这篇论文用了什么参数？我能不能复现实验？有哪些安全或设备限制需要注意？”

2. **展示知识库结构**

   “这个仓库把文档分到 `data/papers/` 和 `data/manuals/`。入库时系统会写入 `doc_type=paper` 或 `doc_type=sop`，这样 Agent 在检索时可以走不同的知识路径。”

3. **运行或说明入库流程**

   “入库流程支持 PDF 和 `.docx` 文件，会解析文档、切分 chunk、生成 embedding、写入 Chroma，并记录处理信息。实际演示时，我可以先放入示例文件，然后运行 `make ingest`。”

4. **打开应用**

   “Streamlit App 提供一个轻量级客户演示界面。侧边栏能看到当前知识库文件，也可以触发入库，或者把回答范围锁定在某一篇论文上。”

5. **提一个论文型问题**

   示例：`这篇论文里的关键制备步骤和参数是什么？`

   “这类问题会偏向 paper path。Agent 会检索论文片段，并在回答中带上引用提示。”

6. **提一个操作型 / 混合型问题**

   示例：`如果我要参考论文复现实验，哪些步骤需要同时核对本实验室 SOP？`

   “这时 Agent 不能只看论文参数，还要考虑 SOP 或手册证据。这个地方正好体现 Demo 的核心价值：区分研究证据和批准操作规范。”

7. **展示 debug 与校验**

   “界面会展示 query analysis、检索引用和 citation validation。这样不仅能给出答案，也能解释 Agent 为什么这样回答。”

8. **总结价值**

   “这个方案的价值是提高技术文档检索效率、增强回答可追溯性，并在涉及操作或安全时保持边界意识。当前版本是本地 Demo，后续可以扩展权限控制、部署、监控和企业知识库集成。”

## 技术架构讲解

RMN Agent 当前可以拆成六层：

- **输入层**：`app.py` 中的 Streamlit chat input，以及可选的 paper scope selector。
- **Query analysis 层**：`query_analyzer.py` 判断问题属于 `SOP_ONLY`、`PAPER_ONLY` 还是 `HYBRID`，并选择回答模式。
- **检索层**：`rag_core.py` 基于 Chroma 检索 paper chunk 和 SOP chunk，支持 metadata filter 与可选的向量 + 词法混合召回。
- **Prompt 编排层**：`fusion_prompts.py` 根据 scholarly、operational、hybrid 场景构造系统 prompt。
- **模型层**：通过 LangChain 调用 Google Gemini 进行 query analysis 和回答生成；embedding provider 可配置。
- **输出与校验层**：Streamlit 流式展示回答和引用，`citation_validator.py` 标记未知引用或缺少引用的数字型声明。

## 业务价值总结

RMN Agent 适合作为售前 Demo，是因为它围绕真实客户关注点组织方案：

- 减少技术文档中的人工检索成本。
- 解释为什么不同类型文档需要不同检索和回答规则。
- 通过引用与校验增强 AI 回答可信度。
- 本地即可演示，不需要先搭建完整生产平台。
- 能清楚展示从原型走向企业级方案的扩展路径。

## 可能被问到的问题与回答

**Q：这是生产级系统吗？**

A：不是。它是一个本地售前 / 作品集 Demo，用于展示架构和可行性。生产使用还需要认证、权限控制、部署打包、日志、监控、更强评估和安全审查。

**Q：它和普通 RAG 聊天机器人有什么区别？**

A：关键区别是知识源分离和意图路由。论文和 SOP 在入库和检索时作为不同文档类型处理，prompt 中也明确要求涉及可执行操作时以 SOP 为规范来源。

**Q：如果 LLM query analyzer 失败怎么办？**

A：`query_analyzer.py` 有保守的 heuristic fallback，会根据关键词、仪器提示、显式文件名和 UI paper anchor 做基本路由和范围判断。

**Q：系统如何降低幻觉？**

A：它要求回答基于检索上下文，事实声明带引用提示，并在生成后做轻量引用校验。这不能证明完全语义忠实，但能捕捉一些常见 Demo 问题，例如引用不存在的来源或数字声明没有引用。

**Q：仓库里包含什么数据？**

A：仓库只包含 `data/` 下的占位目录。真实 Demo 需要自己放入本地非敏感 `.pdf` 或 `.docx` 文件，不应提交客户数据或密钥。

**Q：如果要推进成客户 pilot，优先改什么？**

A：我会优先改上传 / 审核流程、文档级权限、部署打包、更完整评估，以及检索和模型行为的可观测性。
